## Workflow in force (saved as memory: `wordle-ship-fast-prod-dev`)

**Prod is the dev server.** Ship Tier-C frontend straight to prod after local verify —
no per-batch approval needed. GitHub and Cloudflare are **decoupled (no CI auto-deploy)** —
you must deploy manually. Pipeline:

```
npm run typecheck && npm test           # gauntlet
git add <only your files> && git commit
git fetch origin main && git rebase origin/main
git push origin HEAD:main               # never force-push main
npm run deploy                           # wrangler deploy → Cloudflare
# smoke wordul.com, then post a Post-Deploy Summary
```
There's a `/push` skill that does the GitHub+prod sync. Local dev: `npm run dev` (port 8787).

## What shipped this session (in order)

1. **Welcome-back home** — adaptive `#tpl-home`: returning users get a greeting + a
   tappable room list (Recent = owned ∪ played, deduped; Yours = owned), newest-first.
   Two CTAs replaced create-first: **▶ Start playing** (instant solo) and **＋ Invite friend**.
2. **Start playing = instant solo** — `game.autoStart` one-shot fires `{type:"start"}` on the
   first `lobby` snapshot (reconnect-safe, lobby-gated). See `onServerMessage` in app.js.
3. **Invite friend = instant link** — `shareRoomInvite()` fires in the click gesture
   (native share on mobile, clipboard+toast on desktop). Does NOT start a game.
4. **Synced start celebration** — `triggerStartCelebration()`: GO! pop + confetti, fired on
   the `lobby→playing` transition (shared snapshot → simultaneous on all screens). Respects
   Reduced Motion. **Voice is parked** — Yan wants the "golden voice" wishing luck later,
   wired into this same hook.
5. **Two-player mobile layout** — opponents collapse to a compact top strip; YOUR board is
   the full-width hero; **keyboard is `position:sticky;bottom:0`** so it's always reachable.
   Pure CSS (`@media(max-width:520px)`: `.boards{flex-flow:row wrap}` + `order`).
6. **Spectator mini-board fix** — a base `.player-board.spectator .tile{--tile-size:28px}`
   was shadowing the responsive value (custom props resolve at the element that declares
   them). Moved sizing to the board so tiles inherit it.
7. **Shareable result card** — `prepareShareCard()` renders a PNG `<canvas>` (score grid +
   word + green "Beat my score →" pill with the room link) on stats-modal open; cached as a
   `File` so `shareResult()` can `navigator.share({files,url})` synchronously (iOS needs the
   gesture). Desktop fallback: download card + copy link. See `renderResultCanvas()`.
8. **Three new skins** — Arcade (green CRT), Editorial (newsprint), Tactile (warm glass).

## Architecture cheat-sheet

- **`public/app.js`** — single-file SPA. Key fns: `showHome`, `enterNewRoom({autoStart})`,
  `loadHomeRooms`/`buildRoomRows`/`renderRoomList`, `shareRoomInvite`, `triggerStartCelebration`,
  `prepareShareCard`/`renderResultCanvas`/`shareResult`, `onServerMessage` (snapshot → render),
  `showRoom`/`render`/`renderBoards`. `game` object holds client state (`autoStart`, `shareImage`…).
- **`public/style.css`** — all styles. Mobile board layout + sticky keyboard in the
  `@media(max-width:520px)` block. Celebration (`.go-burst`/`.cheer-confetti`). **Edition-scoped
  flourishes** under `html[data-edition="<id>"]` (jackpot/arcade/editorial/tactile).
- **`public/editions/`** — the skin engine (data, not code):
  - `*.js` = an edition `{ id, name, palette, fonts{display,body,link}, motion, sound, companion{lines} }`.
  - `index.js` = `EDITIONS` array + `getEdition(id)`. **Add a skin = new file + one line here.**
  - `../edition.js` = `applyEdition(id)` (maps palette → CSS vars on `<html>`, swaps font link,
    sets `data-edition`, persists `localStorage["wordul.edition"]`), `getActiveEditionId()`,
    `renderEditionPicker(rootEl, onPick)` (chips for ALL editions), gold wallet
    (`getGold/addGold/spendGold/drainGold`), `companionReact(event)`.
  - Current skins: `default`, `yang`, `jackpot`, `arcade`, `editorial`, `tactile`.
  - Decided model: **every design-ritual winner → one new edition.** Free to pick. Hybrid depth
    (palette+type+motion+voice + optional scoped CSS; no layout forks unless truly needed).
- **`public/index.html`** — `#tpl-home`, `#tpl-room` templates; boot script sets `data-edition`
  before first paint to avoid theme flash. The ◆ avatar (top-right) opens the menu that
  currently hosts the edition picker (`renderEditionPicker`, wired in app.js ~avatar menu).
- **Design ritual:** `golden-design-ritual` skill → live prototypes at `wordul.com/designs/<slug>`
  (R2 bucket `wordul-designs`, config `.claude/design-ritual.json`). The 3 room prototypes:
  `/designs/room-neon-arcade`, `/designs/room-editorial-mono`, `/designs/room-tactile-glass`.

## NEXT TASK — Theme selector on the home page

**Yan's ask:** "Select the theme from the home page — show the three latest (or three most
popular) + a 'Browse more themes' [button]." Plus (my add) **preview swatches** so players pick
by look, not name.

### Design notes / pointers
- Add a **theme section to `#tpl-home`** (in `index.html`), likely below the CTAs or under the
  room list. Wire it in `showHome()` (app.js).
- **Which 3:** no popularity tracking exists yet. Cheapest = **3 newest = last 3 of `EDITIONS`**
  (arcade/editorial/tactile today), or add a `featured: true` flag / explicit order in
  `editions/index.js`. "Most popular" would need a usage counter (localStorage tally, or a DO
  endpoint) — recommend shipping "latest/featured" first, add popularity later if wanted.
- **Swatches:** render a mini 3-tile preview per skin from its `palette` (green / yellow / accent)
  — reuse the `palette` object; no need to instantiate the skin. A chip = name + swatch.
- **Apply:** clicking a theme calls `applyEdition(id)` — it restyles the WHOLE app instantly and
  persists. On the home page that doubles as a **live preview** (the home recolors immediately).
  Re-render/refresh any home elements that cache colors if needed.
- **"Browse more themes":** opens the full picker. Reuse `renderEditionPicker(rootEl, onPick)` in
  a modal/sheet (there are existing modal patterns: stats/settings modals in index.html + app.js).
- Keep it free-to-pick (Yan's decision). The gold wallet exists if you ever want paid/unlock skins.
- Verify on mobile (the home is the funnel; swatches must tap well). Then ship per the pipeline.
